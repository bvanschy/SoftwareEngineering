# Testing
## Techniques
Requirements-based, Functional, Black-box

- Equivalence partitioning
- Boundary value analysis
- Robustness testing

Code-based, Structural, White-box

- Control-flow testing
- Data-flow testing

Intuition-based

- Test cases suggested by developer

## Testing Levels

- Unit test
  - Test-driven development
  - Arrange/Act/Assert
  - Code isolation (mocks, dependency injection, IoC containers)
- Integration test
  - Top-down
  - Bottom-up
  - Both
- System test
- Acceptance test

## Test Objectives

- Performance
- Stress, Load
- Error recovery
- Configuration
- Usability
- Installation
- Alpha (testing team within the organization)
- Beta (users external to the organization)
- Regression
